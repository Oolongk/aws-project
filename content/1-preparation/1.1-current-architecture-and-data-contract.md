Tôi chia kiến trúc thành Data Lake path và Serving path. Điểm nối hai path là Lambda QMJ Loader: S3 tạo Parquet mới, S3 gửi `ObjectCreated` event, Lambda đọc file và ghi item sang DynamoDB.

## Data Flow

````text
Data Lake path
Sources → Backfill/Daily Ingestion → S3 Raw → Glue PySpark QMJ → S3 Processed

Serving path
S3 Processed -- ObjectCreated (*.parquet) --> Lambda QMJ Loader --> DynamoDB
DynamoDB --> Lambda Reader --> API Gateway --> Browser

Automation path
EventBridge Scheduler - - -→ Glue Python Shell Daily Ingestion
````

## Data Contract

| Dataset | Grain | Output location | Consumer |
|---|---|---|---|
| `us_prices_raw.parquet` | `ticker + price_date` | `raw_us_market/YYYY/MM/DD/` | Glue PySpark QMJ |
| `us_company_info.parquet` | `ticker + snapshot_date` | `raw_us_market/YYYY/MM/DD/` | Glue PySpark QMJ |
| `us_financials_raw.parquet` | `ticker + report_date + period_type + snapshot_date` | `raw_us_market/YYYY/MM/DD/` | Glue PySpark QMJ |
| QMJ output Parquet | Một row cho một `ticker` tại `AS_OF_DATE` | `processed_qmj/as_of_date=.../` | Lambda QMJ Loader |
| DynamoDB item | `ticker + snapshot_date` | Table `qmj_screener` | Lambda Reader |

> **Tư duy kiến trúc sư – event-driven serving.** `qmj_transform.py` không có `PutItem` hay `BatchWriteItem`. Transform kết thúc ở S3 Processed; Lambda Loader đảm nhận nạp DynamoDB sau khi phát hiện file Parquet mới.
