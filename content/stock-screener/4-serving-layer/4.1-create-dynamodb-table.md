Lambda QMJ Loader ghi một item cho mỗi `ticker` tại mỗi `as_of_date`. Lambda Reader query lịch sử theo ticker, vì vậy table dùng `ticker` làm Partition Key và `snapshot_date` làm Sort Key.

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | AWS Console → DynamoDB → **Tables** | Chọn **Create table** |
| 2 | Table name | Nhập `qmj_screener` để khớp default `TABLE_NAME` trong Lambda Loader |
| 3 | Partition key | Nhập `ticker`, type `String` |
| 4 | Sort key | Tick Add sort key, nhập `snapshot_date`, type `String` |
| 5 | Table settings | Chọn On-demand cho demo/portfolio khi chưa biết traffic |
| 6 | Chọn **Create table** | Chờ Table status thành `Active` |

![Tạo DynamoDB QMJ table](/aws-project/images/stock-screener/create-dynamodb-qmj-table.png?featherlight=false&width=90pc)

## Item contract

````json
{
  "ticker": "AAPL",
  "snapshot_date": "2026-07-06",
  "as_of_date": "2026-07-06",
  "source_s3_key": "processed_qmj/as_of_date=2026-07-06/part-00000.parquet",
  "company_name": "Apple Inc.",
  "qmj_rank": 12,
  "qmj_score": 84.52,
  "qmj_profitability": 90.12,
  "qmj_growth": 75.31,
  "qmj_safety": 88.14,
  "momentum_recent_score": 63.45
}
````

> **Tư duy kiến trúc sư – access pattern first.** `ticker + snapshot_date` cho phép Reader Lambda dùng `Query` để lấy đúng historical series của một mã. Không dùng `Scan` để phục vụ biểu đồ.
