Tạo Role riêng cho Glue, Lambda Loader, Lambda Reader và EventBridge Scheduler. Không hard-code Access Key trong Python source code, Glue argument hoặc Lambda environment variable.

## Role matrix

| Role | Dùng bởi | Quyền tối thiểu |
|---|---|---|
| `AWSGlueServiceRole-StockScreener` | Daily Ingestion và QMJ Transform | Đọc/Ghi Raw, đọc/ghi Processed |
| `LambdaQMJLoaderRole` | Lambda được S3 trigger | List/Get QMJ Parquet, `dynamodb:BatchWriteItem`, CloudWatch Logs |
| `LambdaStockScreenerReaderRole` | Lambda Reader | `dynamodb:Query`, CloudWatch Logs |
| `EventBridgeSchedulerStockScreenerRole` | EventBridge Scheduler | `glue:StartJobRun` cho Daily Ingestion Job |

> **Tư duy kiến trúc sư – boundary per role.** QMJ Transform không cần quyền DynamoDB vì source code chỉ write Processed Parquet. Lambda Loader mới là service có quyền batch write vào `qmj_screener`.

# Tạo Glue Role

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | IAM → Roles → **Create role** | Chọn `AWS service` |
| 2 | Chọn **Glue** | Trust principal là `glue.amazonaws.com` |
| 3 | Đặt Role name | `AWSGlueServiceRole-StockScreener` |
| 4 | Gắn managed policy | `service-role/AWSGlueServiceRole` |
| 5 | Chọn **Create role** | Tạo base role |
| 6 | Role → Permissions → **Add permissions** | Chọn **Create inline policy** |
| 7 | Chọn tab **JSON** | Dán S3 policy bên dưới |
| 8 | Đặt policy name | `StockScreenerGlueS3Access` |
| 9 | Chọn **Create policy** | Hoàn tất |

````json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "ListDataBuckets",
      "Effect": "Allow",
      "Action": ["s3:ListBucket", "s3:GetBucketLocation"],
      "Resource": [
        "arn:aws:s3:::stock-screener-raw-tên-của-bạn",
        "arn:aws:s3:::stock-screener-processed-tên-của-bạn"
      ]
    },
    {
      "Sid": "ReadWriteRawAndProcessedPrefixes",
      "Effect": "Allow",
      "Action": ["s3:GetObject", "s3:PutObject", "s3:DeleteObject"],
      "Resource": [
        "arn:aws:s3:::stock-screener-raw-tên-của-bạn/raw_us_market/*",
        "arn:aws:s3:::stock-screener-processed-tên-của-bạn/processed_qmj/*"
      ]
    }
  ]
}
````

# Tạo Lambda QMJ Loader Role

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | IAM → Roles → **Create role** | Chọn `AWS service` |
| 2 | Chọn **Lambda** | Trust principal là `lambda.amazonaws.com` |
| 3 | Đặt Role name | `LambdaQMJLoaderRole` |
| 4 | Gắn `AWSLambdaBasicExecutionRole` | Lambda ghi logs vào CloudWatch Logs |
| 5 | Chọn **Create role** | Tạo base role |
| 6 | Permissions → **Add permissions** | Chọn **Create inline policy** |
| 7 | Chọn tab **JSON** | Dán S3 + DynamoDB policy dưới đây |
| 8 | Thay `ACCOUNT_ID` | Dùng AWS Account ID của bạn |
| 9 | Đặt policy name | `QMJLoaderS3DynamoDBWrite` |
| 10 | Chọn **Create policy** | Hoàn tất |

````json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "ListProcessedQMJPrefixForBackfill",
      "Effect": "Allow",
      "Action": ["s3:ListBucket"],
      "Resource": "arn:aws:s3:::stock-screener-processed-tên-của-bạn",
      "Condition": {
        "StringLike": {
          "s3:prefix": ["processed_qmj/*"]
        }
      }
    },
    {
      "Sid": "ReadProcessedQMJParquet",
      "Effect": "Allow",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::stock-screener-processed-tên-của-bạn/processed_qmj/*"
    },
    {
      "Sid": "BatchWriteQMJScores",
      "Effect": "Allow",
      "Action": ["dynamodb:BatchWriteItem"],
      "Resource": "arn:aws:dynamodb:ap-southeast-1:ACCOUNT_ID:table/qmj_screener"
    }
  ]
}
````

# Tạo Lambda Reader Role

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | IAM → Roles → **Create role** | Chọn `AWS service` → `Lambda` |
| 2 | Đặt Role name | `LambdaStockScreenerReaderRole` |
| 3 | Gắn `AWSLambdaBasicExecutionRole` | Cho phép ghi CloudWatch Logs |
| 4 | Chọn **Create role** | Tạo base role |
| 5 | Permissions → **Create inline policy** | Dán DynamoDB read policy |
| 6 | Thay `ACCOUNT_ID` | Dùng Account ID của bạn |
| 7 | Đặt policy name | `StockScreenerReaderDynamoDBQuery` |
| 8 | Chọn **Create policy** | Hoàn tất |

````json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "QueryQMJHistoryByTicker",
      "Effect": "Allow",
      "Action": ["dynamodb:Query"],
      "Resource": "arn:aws:dynamodb:ap-southeast-1:ACCOUNT_ID:table/qmj_screener"
    }
  ]
}
````

# Tạo EventBridge Scheduler Execution Role

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | IAM → Roles → **Create role** | Chọn Custom trust policy nếu UI không có Scheduler use case |
| 2 | Trusted service | Dùng `scheduler.amazonaws.com` |
| 3 | Đặt Role name | `EventBridgeSchedulerStockScreenerRole` |
| 4 | Permissions → **Create inline policy** | Cấp quyền start đúng Daily Ingestion Job |
| 5 | Đặt policy name | `SchedulerStartDailyIngestion` |

````json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "StartDailyIngestion",
      "Effect": "Allow",
      "Action": ["glue:StartJobRun"],
      "Resource": "arn:aws:glue:ap-southeast-1:ACCOUNT_ID:job/us-stock-daily-ingestion"
    }
  ]
}
````

````json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {"Service": "scheduler.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }
  ]
}
````

> **Tư duy kiến trúc sư – retry-safe loader.** S3 can deliver duplicate events. Lambda Loader dùng `batch_writer(overwrite_by_pkeys=["ticker", "snapshot_date"])`, nên ghi lại cùng một ticker/ngày sẽ overwrite item thay vì tạo duplicate logical record.
