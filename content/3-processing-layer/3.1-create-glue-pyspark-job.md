QMJ Transform là Spark ETL Job đọc toàn bộ Raw Parquet bằng `recursiveFileLookup` và ghi output theo `as_of_date`.

## Upload transform script

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | Amazon S3 → `stock-screener-raw-tên-của-bạn/glue-scripts/` | Dùng script prefix hiện có |
| 2 | Chọn **Upload** | Upload `qmj_transform.py` |
| 3 | Kiểm tra S3 URI | `s3://stock-screener-raw-tên-của-bạn/glue-scripts/qmj_transform.py` |

## Tạo Spark Job

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | AWS Glue → ETL jobs → **Create job** | Mở wizard tạo Job |
| 2 | Chọn job type | Chọn **Spark** |
| 3 | Nhập Job name | `us-stock-qmj-transform` |
| 4 | Chọn IAM Role | `AWSGlueServiceRole-StockScreener` |
| 5 | Chọn Script path | URI của `qmj_transform.py` |
| 6 | Chọn Glue runtime | Dùng Spark runtime đã test với project |
| 7 | Chọn capacity | Bắt đầu nhỏ, tăng sau khi xem duration và memory |
| 8 | Mở **Job parameters** | Nhập args ở bảng bên dưới |
| 9 | Chọn **Save** | Lưu job |
| 10 | Chọn **Run** | Test `AS_OF_DATE` có Raw data |

## Job parameters

| Key | Value mẫu | Source sử dụng |
|---|---|---|
| `--RAW_BUCKET` | `stock-screener-raw-tên-của-bạn` | Override Raw Bucket |
| `--RAW_PREFIX` | `raw_us_market` | Override Raw prefix |
| `--PROCESSED_BUCKET` | `stock-screener-processed-tên-của-bạn` | Override Processed Bucket |
| `--PROCESSED_PREFIX` | `processed_qmj` | Override Processed prefix |
| `--AS_OF_DATE` | `2026-07-03` | Cut-off date để lấy info, prices, financial reports |
| `--LOOKBACK_DAYS` | `800` | Lookback calendar days cho price history |
| `--MIN_PRICE_OBSERVATIONS` | `200` | Đang parse nhưng chưa được filter trong source hiện tại |
````text
Trong Glue Console, key phải bắt đầu bằng hai dấu gạch ngang.
Ví dụ: --AS_OF_DATE
````

> **Tư duy kiến trúc sư – parameterized rerun.** `AS_OF_DATE` giúp bạn rerun lịch sử và điều tra output từng ngày mà không cần sửa code. Đây là nền tảng cho reproducibility của processing layer.
