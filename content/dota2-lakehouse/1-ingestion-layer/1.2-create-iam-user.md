Để Python script có thể đẩy dữ liệu lên S3, nó cần một bộ chìa khóa (Access Key). Chúng ta sẽ tạo một IAM User và chỉ cấp đúng quyền ghi file vào Raw Bucket.

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | Truy cập IAM | Tìm dịch vụ **IAM** > Chọn **Users** > Nhấn **Create user**. |
| 2 | Nhập thông tin User | User name: `dota2-ingestion-bot`. Bỏ qua tùy chọn *Provide user access to the AWS Management Console*. Nhấn **Next**. |
| 3 | Khởi tạo Policy | Ở màn hình Permissions, chọn **Attach policies directly** > Nhấn **Create policy** (hệ thống sẽ mở tab mới). |
| 4 | Cấu hình Policy (JSON) | Chuyển sang định dạng **JSON** và dán đoạn mã phân quyền bên dưới vào > Nhấn **Next**. |
| 5 | Lưu Policy | Tên Policy: `S3_PutObject_Dota2_Raw`. Nhấn **Create policy**. |
| 6 | Gắn Policy cho User | Quay lại tab tạo User, nhấn nút *Refresh*, tìm và tích chọn `S3_PutObject_Dota2_Raw` > Nhấn **Next** > **Create user**. |
| 7 | Sinh Access Key | Click vào User vừa tạo > Tab **Security credentials** > Nhấn **Create access key** > Chọn *Local code*. Tải file `.csv` chứa cặp Access Key và Secret Key về máy. |

Mã JSON Policy (Nhớ thay thế phần `YOUR_BUCKET_NAME` bằng tên bucket bạn vừa tạo ở bài 1.1):

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "s3:PutObject",
            "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
        }
    ]
}
```
