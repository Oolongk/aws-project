Chúng ta tiến hành nâng cấp mã nguồn PySpark của AWS Glue Job. Ngoài nhiệm vụ lưu trữ file Parquet xuống S3, Glue sẽ tính toán trực tiếp tỷ lệ thắng (Winrate), tỷ lệ chọn (Pickrate) và dùng thư viện `boto3` để đẩy thẳng kết quả vào DynamoDB.

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | Mở Glue Studio | Vào AWS Console > Tìm dịch vụ **AWS Glue** > Mục **ETL Jobs** > Chọn **Visual ETL** hoặc mở trực tiếp mã nguồn của Job `dota2_etl_job` đã tạo ở bài 2.3. |
| 2 | Cập nhật Script | Chọn tab **Script**, xóa toàn bộ mã nguồn cũ đi và thay thế bằng đoạn code PySpark nâng cấp ở bên dưới. |
| 3 | Cấu hình tham số | Đảm bảo các biến `RAW_PATH`, `PROCESSED_PATH` và tên bảng DynamoDB đã khớp với hệ thống của bạn. |
| 4 | Lưu và Chạy | Nhấn **Save** > Nhấn **Run** để khởi chạy pipeline tính toán tổng hợp. |
| 5 | Kiểm duyệt kết quả | Truy cập **DynamoDB Console** > Mục **Tables** > Click vào bảng `dota2_hero_metrics` > Nhấn nút **Explore table items** để kiểm tra các bản ghi metrics của các vị Tướng đã được nạp thành công. |

**Mã nguồn PySpark nâng cấp (`dota2_etl_job_v2.py`):**

```python
import sys
import boto3
from pyspark.context import SparkContext
from awsglue.context import GlueContext
from pyspark.sql.functions import explode, col, from_unixtime, to_date, count, sum as _sum, when, lit

sc = SparkContext.getOrCreate()
glueContext = GlueContext(sc)
spark = glueContext.spark_session

RAW_PATH = "s3://dota2-raw-zone-tencuaban/raw_matches/"
PROCESSED_PATH = "s3://dota2-processed-zone-tencuaban/matches_parquet/"
DYNAMODB_TABLE = "dota2_hero_metrics"

# 1. Đọc dữ liệu thô và Flattening
df = spark.read.json(RAW_PATH)
df_exploded = df.select(
    col("match_id"),
    to_date(from_unixtime(col("start_time"))).alias("match_date"),
    explode(col("players")).alias("player")
)
df_final = df_exploded.select(
    "match_id",
    "match_date",
    col("player.hero_id").alias("hero_id"),
    col("player.win").alias("is_win")
)

# [Tối ưu Chương 2]: Ghi dữ liệu sạch lưu vết dài hạn dưới dạng Parquet Snappy xuống S3
df_final.write.mode("append").partitionBy("match_date").parquet(PROCESSED_PATH, compression="snappy")

# 2. [Nâng cấp Chương 4]: Tính toán Aggregation Metrics bằng Spark DataFrame API
print("Đang tính toán tổng hợp tỷ lệ Winrate và Pickrate của các Hero...")
total_matches = df.count()

df_metrics = df_final.groupBy("hero_id").agg(
    count("match_id").alias("total_picks"),
    _sum(when(col("is_win") == True, 1).otherwise(0)).alias("total_wins")
)

# Tính toán tỷ lệ phần trăm dữ liệu
df_metrics = df_metrics.withColumn("win_rate", (col("total_wins") / col("total_picks")) * 100) \
                       .withColumn("pick_rate", (col("total_picks") / lit(total_matches)) * 100)

# 3. Đẩy dữ liệu sang DynamoDB sử dụng boto3 kết hợp với phân vùng tính toán của Spark
def upload_row_to_dynamodb(row):
    # Khởi tạo boto3 client bên trong từng Worker Node của Spark Cluster
    dynamodb = boto3.resource('dynamodb', region_name='ap-southeast-1')
    table = dynamodb.Table(DYNAMODB_TABLE)
    
    # Thực hiện hành động Upsert dữ liệu (Ghi mới hoặc cập nhật nếu trùng hero_id)
    table.put_item(
        Item={
            'hero_id': int(row['hero_id']),
            'total_picks': int(row['total_picks']),
            'total_wins': int(row['total_wins']),
            'win_rate': str(round(row['win_rate'], 2)), # Ép sang chuỗi để tránh lỗi định dạng Float trong DynamoDB
            'pick_rate': str(round(row['pick_rate'], 2))
        }
    )

# Gọi hàm thực thi phân tán trên toàn bộ các dòng của DataFrame
print("Bắt đầu nạp dữ liệu phân tán vào Amazon DynamoDB...")
df_metrics.rdd.foreach(upload_row_to_dynamodb)
print("Nạp dữ liệu hoàn tất!")
