Thông thường, mọi người dùng **AWS Glue Crawler** để tự động dò tìm cấu trúc file và tạo bảng. Nhưng Crawler chạy chậm và tốn tiền. Thay vào đó, chúng ta sẽ viết thẳng một câu lệnh DDL (Data Definition Language) có nhúng kỹ thuật **Partition Projection** để tạo bảng. Kỹ thuật này giúp Athena tính toán phân vùng siêu nhanh mà không cần Crawler.

| Bước | Hành động | Chi tiết kỹ thuật |
|---|---|---|
| 1 | Mở Query Editor | Trong giao diện Athena, chuyển sang tab **Query editor**. |
| 2 | Chạy DDL | Copy đoạn mã SQL bên dưới, dán vào khung truy vấn (thay đường dẫn `LOCATION` bằng S3 Processed Bucket của bạn) và nhấn **Run**. |
| 3 | Truy vấn thử | Xóa lệnh cũ đi, gõ lệnh mới: `SELECT * FROM dota2_matches LIMIT 10;` và nhấn **Run** để chiêm ngưỡng thành quả. |

**Đoạn mã DDL SQL tích hợp Partition Projection:**

```sql
CREATE EXTERNAL TABLE IF NOT EXISTS dota2_matches (
  `match_id` BIGINT,
  `account_id` BIGINT,
  `hero_id` INT,
  `is_win` BOOLEAN
)
PARTITIONED BY (
  `match_date` STRING
)
STORED AS PARQUET
LOCATION 's3://dota2-processed-zone-tencuaban/matches_parquet/'
TBLPROPERTIES (
  'projection.enabled' = 'true',
  'projection.match_date.type' = 'date',
  'projection.match_date.range' = '2025-01-01,NOW',
  'projection.match_date.format' = 'yyyy-MM-dd',
  'projection.match_date.interval' = '1',
  'projection.match_date.interval.unit' = 'DAYS',
  'storage.location.template' = 's3://dota2-processed-zone-tencuaban/matches_parquet/match_date=${match_date}/'
);
