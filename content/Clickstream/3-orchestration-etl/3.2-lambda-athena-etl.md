Bài viết này hướng dẫn bạn cấu hình khối xử lý dữ liệu lõi (Core ETL Compute Layer). Chúng ta sẽ viết một đoạn script Python thực thi trên nền AWS Lambda để điều khiển dịch vụ Amazon Athena chạy truy vấn SQL Serverless, ép kiểu dữ liệu Parquet và bóc tách mốc mốc thời gian về dạng số nguyên nguyên bản cho AI.

#### Hướng dẫn cấu hình hàm tính toán AWS Lambda

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Khởi tạo hàm | Vào dịch vụ **AWS Lambda** -> Bấm **Create function** -> Chọn **Author from scratch**. |
| **2** | Thiết lập cơ bản | Đặt tên hàm là `AthenaETLProcessor`. Chọn Runtime ngôn ngữ là **Python 3.12**. |
| **3** | Cấu hình thời gian | Chuyển sang tab **Configuration** -> mục **General configuration** -> Bấm Edit, tăng thông số **Timeout** lên mức tối thiểu là `2 minutes` (để tránh hàm bị ngắt giữa chừng khi đợi Athena kết xuất file). |
| **4** | Cấp quyền hệ thống | Tại mục **Permissions**, bấm vào link Execution Role để mở giao diện IAM. Tiến hành đính kèm thêm 2 chính sách quyền chuẩn: `AmazonAthenaFullAccess` và `AmazonS3FullAccess`. |

#### Mã nguồn xử lý ETL (Python & Serverless SQL)

Bạn hãy quay lại tab **Code** của hàm Lambda, xóa sạch đoạn mã mặc định, dán toàn bộ khối mã nguồn chuẩn Enterprise dưới đây vào và bấm nút **Deploy** màu cam:

```python
import boto3
import time

def lambda_handler(event, context):
    athena_client = boto3.client('athena')
    
    # Định nghĩa cấu hình cơ sở dữ liệu và thư mục đích
    DATABASE = 'clickstream_db'
    # ⚠️ Lưu ý: Thay chính xác tên S3 Bucket Data Lake của bạn vào dòng dưới đây
    OUTPUT_LOCATION = 's3://your-clickstream-datalake-bucket-name/personalize_data/' 
    
    # Câu lệnh SQL bóc tách Parquet, viết hoa tiêu đề cột và chuyển đổi định dạng thời gian sang UNIX Epoch Time
    query = """
    SELECT 
        user_id AS USER_ID,
        page_url AS ITEM_ID,
        CAST(to_unixtime(CAST(event_timestamp AS timestamp)) AS bigint) AS "TIMESTAMP",
        event_type AS EVENT_TYPE,
        device_os AS DEVICE_OS
    FROM "clickstream_db"."crawled_data"
    WHERE user_id IS NOT NULL;
    """
    
    print("🚀 Đánh thức Amazon Athena thực thi truy vấn SQL Serverless ETL...")
    response = athena_client.start_query_execution(
        QueryString=query,
        QueryExecutionContext={'Database': DATABASE},
        ResultConfiguration={'OutputLocation': OUTPUT_LOCATION}
    )
    
    query_execution_id = response['QueryExecutionId']
    
    # Kỹ thuật Polling Vòng lặp kiểm tra trạng thái bất đồng bộ của Athena
    state = 'RUNNING'
    while state in ['RUNNING', 'QUEUED']:
        check_status = athena_client.get_query_execution(QueryExecutionId=query_execution_id)
        state = check_status['QueryExecution']['Status']['State']
        if state in ['RUNNING', 'QUEUED']:
            time.sleep(2)
            
    if state == 'SUCCEEDED':
        print(f"✅ Xuất sắc! Dữ liệu CSV chuẩn hóa cho AI đã hạ cánh an toàn tại: {OUTPUT_LOCATION}")
        return {'statusCode': 200, 'body': 'ETL Biến Đổi Dữ Liệu Thành Công!'}
    else:
        raise Exception(f"❌ Đường ống Athena ETL gục ngã tại trạng thái lỗi: {state}")
```
#### Kiểm tra kết quả thực thi 
Sau khi luồng xử lý này chạy thành công thông qua lệnh điều phối của Step Functions, bạn hãy mở Amazon S3, vào thư mục personalize_data/. Bạn sẽ thấy một file dạng .csv mới tinh được tạo ra. Khi mở file, mốc thời gian thô dạng chữ (2026-06-25 13:00:00) đã được dọn dẹp sạch sẽ và chuyển đổi thành dãy số nguyên máy tính hoàn hảo (Ví dụ: 1782306000). Bạn đã hoàn tất 100% hạ tầng Data Pipeline để chuẩn bị bước sang chương nạp dữ liệu cho AI!
