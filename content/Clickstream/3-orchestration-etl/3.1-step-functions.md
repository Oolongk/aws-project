Để giải phóng hệ thống khỏi các tác vụ kích hoạt bằng tay, chúng ta sẽ sử dụng **AWS Step Functions Workflow Studio** nhằm xây dựng một quy trình tự động hóa hoàn toàn theo chuỗi tuần tự.

#### Bước 1: Khởi tạo AWS Glue Crawler để quét dữ liệu

Trước khi Step Functions có thể điều phối, chúng ta cần một công cụ tự động nhận diện các file Parquet mới hạ cánh xuống S3 nhằm cập nhật sổ cái (Data Catalog).

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Tìm kiếm dịch vụ | Tại thanh tìm kiếm AWS Console, gõ **Glue** và chọn **Crawlers**. |
| **2** | Tạo Crawler | Bấm nút **Create crawler**, đặt tên là `Clickstream_S3_Crawler`. |
| **3** | Thêm nguồn dữ liệu | Bấm **Add a data source**, chọn S3 và trỏ đường dẫn tuyệt đối về thư mục dữ liệu thô: `s3://your-bucket-name/data/`. |
| **4** | Cấu hình quyền IAM | Chọn **Create new IAM Role** để AWS tự cấp quyền đọc S3 cho Crawler. |
| **5** | Chỉ định Database | Tại mục Output configuration, tạo một Database mới tên là `clickstream_db`. |
| **6** | Hoàn tất | Bấm **Create crawler** để lưu cấu hình. |

#### Bước 2: Thiết lập chuỗi State Machine chống Race Condition

Chúng ta tiến hành lắp ráp các dịch vụ thành một chuỗi tự động thông qua giao diện kéo thả trực quan của Step Functions.

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Tạo State Machine | Vào dịch vụ **Step Functions** -> Bấm **Create state machine** -> Chọn **Workflow Studio** (Kéo thả). |
| **2** | Kéo khối Crawler | Ở bộ lọc bên trái, gõ `Glue` -> Kéo khối **Glue StartCrawler** và thả vào điểm bắt đầu (`Start`). Tại cột cấu hình bên phải, nhập tên crawler: `Clickstream_S3_Crawler`. |
| **3** | Chèn Trạng thái Chờ | Ở bộ lọc bên trái, gõ `Wait` -> Kéo khối **Wait State** (hình đồng hồ) thả vào **NGAY SAU** khối Crawler. Thiết lập thời gian là **120 Seconds**. |
| **4** | Kéo khối Lambda | Ở bộ lọc bên trái, gõ `Lambda` -> Kéo khối **Lambda Invoke** thả vào sau khối Wait. Trỏ hàm mục tiêu về hàm `AthenaETLProcessor` (sẽ tạo ở bài 3.2). |
| **5** | Kéo khối SNS | Ở bộ lọc bên trái, gõ `SNS` -> Kéo khối **SNS Publish** thả vào cuối luồng xử lý. Chọn Topic ARN trỏ về `Pipeline_Completion_Alert` đã chuẩn bị ở Chương 1. |
| **6** | Lưu và Triển khai | Bấm **Save** ở góc trên bên phải để lưu lại sơ đồ Nhạc trưởng. |

> **Cảnh báo Lỗi hệ thống (Race Condition):** Việc bổ sung khối **Wait State (120 giây)** là một kỹ nghệ hạ tầng bắt buộc. AWS Glue Crawler cần trung bình từ 1 đến 2 phút để quét đĩa cứng S3 và cập nhật cấu trúc bảng vào Data Catalog thành công. Trong khi đó, robot tính toán AWS Lambda hoạt động rất nhanh (chỉ mất 3 giây). Nếu không có khối chặn Wait State, Lambda sẽ lao vào thực thi câu lệnh SQL ngay lập tức lúc sổ cái dữ liệu đang trống, dẫn đến tệp CSV đầu ra của bạn bị rỗng hoàn toàn.

#### Bước 3: Đặt lịch chạy tự động bằng Amazon EventBridge Scheduler

Giao diện AWS hiện tại đã tách biệt tính năng lập lịch thành một công cụ chuyên dụng mạnh mẽ có tên là **EventBridge Scheduler**. Chúng ta sẽ sử dụng công cụ này để đánh thức Nhạc trưởng.

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Truy cập Scheduler | Vào dịch vụ **Amazon EventBridge** -> Nhìn sang menu bên trái, tìm mục **Scheduler** và chọn **Schedules**. |
| **2** | Tạo Lịch trình | Bấm nút **Create schedule**. |
| **3** | Đặt tên | Tại ô Schedule name, nhập tên là `Trigger-NewsPipeline-Hourly`. |
| **4** | Cấu hình thời gian | Tại mục *Schedule pattern*, chọn **Recurring schedule** (Lịch lặp lại). Tiếp tục chọn **Rate-based schedule**, nhập số `1` và đơn vị là `Hours` (Hệ thống tự động chạy 1 giờ/lần). Cuộn xuống và bấm Next. |
| **5** | Chỉ định Target | Tại trang Target detail, tìm và chọn **AWS Step Functions** -> Chọn API **StartExecution**. Tại ô *State machine*, trỏ vào tên Nhạc trưởng bạn vừa vẽ ở Bước 2. Cuộn xuống bấm Next. |
| **6** | Phân quyền & Hoàn tất | Ở trang Settings, tại mục *Permissions*, giữ nguyên tùy chọn **Create new role for this schedule** để AWS tự động cấp quyền. Bấm **Create schedule**. |

#### Kiểm tra kết quả
Kể từ giây phút này, hệ thống của bạn đã đạt cảnh giới tự vận hành 100%. Cứ đúng chu kỳ 1 tiếng, EventBridge Scheduler sẽ đánh thức Step Functions dậy, chạy khép kín toàn bộ dây chuyền xử lý dữ liệu lớn mà bạn không cần phải bấm bất kỳ một nút Execute thủ công nào nữa.
