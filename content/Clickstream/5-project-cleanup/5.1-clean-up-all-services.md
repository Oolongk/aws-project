Hãy thực hiện dọn dẹp theo đúng thứ tự các bảng dưới đây. Tuyệt đối không xóa S3 Data Lake trước khi xóa các dịch vụ khác, vì nó sẽ gây ra lỗi nghẽn hệ thống.

#### 1. Xóa Tầng Machine Learning (Ưu tiên cao nhất)

Đây là thành phần "đốt tiền" nhanh nhất. Bạn bắt buộc phải dọn dẹp nó đầu tiên.

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Xóa Campaign | Truy cập **Amazon Personalize** -> Chọn **Campaigns** -> Bấm **Delete**. (Nếu bạn đã áp dụng chiến thuật FinOps "Hit and Run" ở chương trước, hãy bỏ qua bước này). |
| **2** | Xóa Solution | Chọn mục **Solutions** -> Mở Solution của bạn -> Bấm **Delete**. |
| **3** | Xóa Dataset | Mở mục **Datasets** -> Lần lượt xóa tất cả các Dataset bên trong. |
| **4** | Xóa Dataset Group | Cuối cùng, chọn **Dataset groups** -> Chọn nhóm `news_recommender_group` và bấm **Delete**. |

> **Lưu ý:** Quá trình xóa Solution có thể mất vài phút. Bạn phải đợi Solution biến mất hoàn toàn thì mới có thể xóa được Dataset Group.

---

#### 2. Xóa Tầng Orchestration và Compute

Gỡ bỏ các bộ điều phối và hàm tính toán để dừng hoàn toàn dòng chảy tự động của hệ thống.

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Tắt Báo thức | Truy cập **Amazon EventBridge** -> Chọn **Schedules** -> Chọn lịch trình `Trigger-NewsPipeline-Hourly` -> Bấm **Delete**. |
| **2** | Xóa Nhạc trưởng | Truy cập **AWS Step Functions** -> Chọn State Machine của bạn -> Bấm **Delete**. |
| **3** | Xóa Hàm ETL | Truy cập **AWS Lambda** -> Chọn hàm `AthenaETLProcessor` -> Bấm **Delete**. |

---

#### 3. Xóa Tầng Data Ingestion & Storage

Dọn dẹp các đường ống Streaming và kho lưu trữ vật lý.

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Xóa Data Catalog | Truy cập **AWS Glue**. Vào **Crawlers** để xóa Crawler. Vào **Databases** để xóa `clickstream_db` (Thao tác này sẽ xóa luôn các bảng bên trong). |
| **2** | Xóa Streaming | Truy cập **Kinesis Data Firehose** -> Chọn luồng Delivery Stream -> Bấm **Delete**. |
| **3** | Dọn dẹp Data Lake | Truy cập **Amazon S3** -> Chọn bucket `clickstream-datalake-v2...`. Bạn **BẮT BUỘC** phải bấm nút **Empty** (để xóa sạch các file Parquet/CSV bên trong) trước. Sau khi Empty thành công, chọn lại bucket đó và bấm **Delete**. |
| **4** | Xóa Log Athena | Vẫn ở **Amazon S3**, tìm bucket có tên bắt đầu bằng `aws-athena-query-results-...` -> Tiến hành **Empty** và **Delete** tương tự. |
| **5** | Xóa Thông báo | Truy cập **Amazon SNS** -> Chọn Topics -> Xóa topic `Pipeline_Completion_Alert`. |

---

#### 4. Dọn dẹp Bảo mật (IAM Credentials & Roles)

Đây là bước cực kỳ quan trọng để bảo vệ tài khoản AWS của bạn khỏi rủi ro bảo mật sau khi thực hành xong.

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Xóa User Giả lập | Truy cập **IAM** -> Chọn **Users** -> Tích chọn User `Clickstream-Data-Pusher` mà bạn đã tạo ở Chương 2 -> Bấm **Delete** để xóa hoàn toàn tài khoản phụ này (Access Key đi kèm cũng sẽ bị hủy vĩnh viễn theo User). |
| **2** | Dọn dẹp IAM Roles | (Tùy chọn) Chuyển sang mục **Roles**, tìm và xóa các Role tự động được tạo ra bởi hệ thống (thường chứa cụm từ *Personalize, Firehose, StepFunctions* hoặc *Glue*). |
