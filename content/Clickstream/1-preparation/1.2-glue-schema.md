Để Kinesis Firehose có thể nén dữ liệu từ dạng JSON phi cấu trúc sang định dạng cột siêu nén Apache Parquet ngay trên luồng chạy, nó cần một "bản thiết kế" để biết dữ liệu gồm những cột nào. Chúng ta sẽ giải quyết bài toán này bằng **AWS Glue Data Catalog** (Bao gồm Database và Crawler).

#### Bước 1: Tạo Cơ sở dữ liệu (Database)

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Truy cập dịch vụ | Tại AWS Console, tìm kiếm và mở **AWS Glue**. |
| **2** | Vào danh mục Data Catalog | Ở menu bên trái, chọn **Databases** và bấm **Add database**. |
| **3** | Khai báo tên | Nhập tên `clickstream_db` và bấm **Create database**. |

#### Bước 2: Tạo Bảng cấu trúc bằng Crawler (Kỹ thuật Schema-on-Read)

Để có một bảng (Table) chuẩn xác đưa cho Firehose, cách thông minh nhất là cho một con bọ cạp (Crawler) đọc thử một file JSON mẫu.

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Tạo file mẫu (Dummy file) | Tạo 1 file `sample.json` trên máy tính chứa nội dung: `{"user_id": "1", "event_type": "view", "event_timestamp": "2026", "page_url": "/home", "device_os": "iOS"}`. |
| **2** | Upload lên S3 | Tải file này lên một thư mục tạm trên S3 (ví dụ: `s3://clickstream-datalake-v2/schema-sample/`). |
| **3** | Tạo Crawler | Quay lại AWS Glue, chọn **Crawlers** -> Bấm **Create crawler** (Tên: `Schema_Crawler`). |
| **4** | Trỏ nguồn dữ liệu | Chọn Add a data source và trỏ đường dẫn S3 về đúng thư mục chứa file `sample.json` ở trên. |
| **5** | Phân quyền và Đầu ra | Cấp quyền IAM Role, chọn Database đích là `clickstream_db` và bấm **Run Crawler**. |

> **Tư duy Kỹ thuật:** Sau khi Crawler chạy xong, nó sẽ tự động sinh ra một Bảng (Table) trong Database `clickstream_db` chứa đầy đủ 5 cột tương ứng với file JSON. Việc dùng Crawler đọc file mẫu là thủ thuật (Best Practice) giúp kỹ sư Data tránh việc phải gõ tay định nghĩa từng cột một cách thủ công, loại bỏ hoàn toàn rủi ro sai lỗi chính tả cấu trúc.
