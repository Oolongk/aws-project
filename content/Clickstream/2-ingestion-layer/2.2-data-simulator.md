Để hệ thống có dữ liệu đầu vào phục vụ cho việc huấn luyện AI, tôi đã xây dựng một trình giả lập dữ liệu Clickstream bằng Python. Trình giả lập này mô phỏng hành vi của 3 nhóm người dùng (Persona) với trọng số khác nhau, tạo ra tập dữ liệu có quy luật thực tế.

#### Bước 0: Tạo User chuyên dụng và Lấy khóa bảo mật (Best Practice)

Để đảm bảo an toàn tuyệt đối cho hệ thống (Nguyên tắc quyền hạn tối thiểu - Principle of Least Privilege), chúng ta sẽ KHÔNG sử dụng tài khoản Admin. Thay vào đó, tôi sẽ tạo một IAM User ẩn danh chuyên dụng chỉ có quyền đẩy dữ liệu lên Kinesis Firehose.

| Bước | Hành động | Chi tiết kỹ thuật |
| :--- | :--- | :--- |
| **1** | Truy cập IAM | Tại AWS Console, gõ tìm kiếm **IAM** -> Chọn **Users** -> Bấm **Create user**. |
| **2** | Định danh User | Đặt tên User là `Clickstream-Data-Pusher` và bấm Next. |
| **3** | Phân quyền (Least Privilege) | Chọn **Attach policies directly**. Tìm và tích chọn chính sách `AmazonKinesisFirehoseFullAccess`. Bấm Next -> Create user. |
| **4** | Mở tab bảo mật | Click vào tên user `Clickstream-Data-Pusher` vừa tạo, chuyển sang tab **Security credentials**. |
| **5** | Tạo khóa mới | Cuộn xuống phần **Access keys**, bấm nút **Create access key**. Chọn **Local code** (Mã cục bộ), xác nhận và bấm Next. |
| **6** | Lưu trữ khóa | Màn hình hiện ra `Access key` và `Secret access key`. Hãy copy hai chuỗi này để dán vào code Python bên dưới. |

> **Cảnh báo Bảo mật (Security Alert):** Chuỗi `Secret access key` chỉ hiển thị ĐÚNG MỘT LẦN duy nhất ở bước này. Tuyệt đối không chia sẻ hoặc tải bộ khóa này lên các kho lưu trữ mã nguồn mở (như GitHub Public) để tránh bị rò rỉ tài khoản. Hãy xóa bộ khóa này trên giao diện IAM ngay sau khi hoàn thành dự án.

#### Mã nguồn trình giả lập (Python)

Đây là đoạn code cốt lõi tạo ra các sự kiện Clickstream phân phối theo Persona. Mã nguồn đã được tích hợp khối bẫy lỗi để dừng an toàn:

```python
import boto3
import json
import time
import random
import datetime

# ----------------------------------------------------------------------
# [CẢNH BÁO BẢO MẬT] 
# Nhớ xóa Key sau khi hoàn thành dự án để tránh mất tiền oan nhé!
# ----------------------------------------------------------------------
ACCESS_KEY = 'YOUR_AWS_ACCESS_KEY'
SECRET_KEY = 'YOUR_AWS_SECRET_KEY'
REGION = 'ap-southeast-1'
STREAM_NAME = 'clickstream-delivery-stream' 

# Khởi tạo kết nối trực tiếp với Kinesis Firehose
client = boto3.client(
    'firehose',
    region_name=REGION,
    aws_access_key_id=ACCESS_KEY,
    aws_secret_access_key=SECRET_KEY
)

def generate_news_persona_event():
    # Thuật toán AI: Chia tỷ lệ 3 nhóm độc giả báo chí (30% Tech, 30% Finance, 40% GenZ)
    persona = random.choices(['Tech_Bro', 'Finance_Investor', 'GenZ_Fan'], weights=[0.3, 0.3, 0.4])[0]
    
    if persona == 'Tech_Bro':
        # Nhóm 1: Dân Công nghệ - Chỉ dùng MacOS/Windows, thích đọc tin Tech/AI
        user_id = str(random.randint(1000, 1999))
        device = random.choices(['MacOS', 'Windows'], weights=[0.6, 0.4])[0]
        url = random.choices(['/tech/ai-chatgpt', '/startup/kydang', '/tech/aws-cloud'], weights=[0.5, 0.2, 0.3])[0]
        
    elif persona == 'Finance_Investor':
        # Nhóm 2: Dân Đầu tư - Dùng nhiều iOS, thích đọc tin Chứng khoán/Bất động sản
        user_id = str(random.randint(2000, 2999))
        device = random.choices(['iOS', 'Windows'], weights=[0.8, 0.2])[0]
        url = random.choices(['/finance/stock-market', '/real-estate/vinhomes'], weights=[0.6, 0.4])[0]
        
    else:
        # Nhóm 3: Gen Z Giải trí - Dùng Android/iOS, thích tin Showbiz/K-pop
        user_id = str(random.randint(3000, 3999))
        device = random.choices(['Android', 'iOS'], weights=[0.7, 0.3])[0]
        url = random.choices(['/entertainment/kpop-news', '/music/youtube-trending'], weights=[0.6, 0.4])[0]

    # Đóng gói dữ liệu chuẩn Schema của AWS Glue
    return {
        'user_id': user_id,
        'event_type': 'page_view', # Hành vi đọc báo chủ yếu là xem trang
        'event_timestamp': datetime.datetime.now().strftime('%Y-%m-%d %H:%M:%S'), 
        'page_url': url,
        'device_os': device
    }

print("=========================================================")
print(f"🚀 Bắt đầu phát sóng dữ liệu Báo Chí (Persona) vào {STREAM_NAME}...")
print("=========================================================\n")

events_sent = 0

try:
    while True:
        event = generate_news_persona_event()
        # Biến Dict thành JSON string, thêm \n để Firehose cắt dòng chuẩn xác
        data_as_string = json.dumps(event) + '\n'
        
        # Bắn dữ liệu lên AWS
        client.put_record(
            DeliveryStreamName=STREAM_NAME,
            Record={'Data': data_as_string}
        )
        
        events_sent += 1
        
        # In log ra màn hình cho đẹp và dễ theo dõi
        print(f"[Log] Đã gửi: Độc giả {event['user_id']} ({event['device_os']}) đang đọc {event['page_url']}")
        
        # Dừng 0.1s (Tương đương 10 sự kiện/giây) - Đủ để tích data mà không tốn nhiều dung lượng
        time.sleep(0.1) 

except KeyboardInterrupt:
    print(f"\n🛑 Đã dừng hệ thống! Tổng cộng có {events_sent} lượt đọc báo được gửi lên Data Lake.")
